# Governor：Mac 电源模式自动切换软件方案

## 1. 任务、范围与交付边界

实现一个 macOS 菜单栏工具，仅根据**用户空闲时间**和**最近 15 秒平均 CPU 使用率**，在 Low Power、Automatic 和 High Power 之间自动选择电源模式。

首版只解决以下四件事：

1. 用户长时间无操作且 CPU 负载不高时，进入 Low Power。
2. CPU 负载较高时，保持活跃；仅在当前环境实际支持时进入 High Power。
3. 用户恢复操作后，立即退出 Low Power，并重新选择 Automatic 或 High Power。
4. 用户可选择让软件在自动进入 Low Power 前保存内建屏幕亮度，并在确认退出后按设定等待时间尽力恢复该亮度。
5. 用户可在设置中选择 `English` 或 `中文` 界面语言。

本方案中的“实现”仅限上述行为、列出的菜单栏内容、错误处理、状态恢复和验收测试。不得自行增加电量条件、应用名单、多套 CPU 阈值、定时任务、智能学习、通知策略、复杂诊断设置或其他用户可见功能。

## 2. 术语与事实来源

| 术语 | 定义 |
|---|---|
| CPU 使用率 | 每次决策时使用最近 15 秒的平均值。首版固定，不提供设置。 |
| 高负载 | CPU **严格大于 60%**；60.0% 不属于高负载。 |
| 已达到空闲时间 | macOS 报告的连续无操作时长不小于用户设置的空闲时间。 |
| High Power 可用 | 必须根据当前 Mac 与当前电源状态实际检测；不得通过机型名称推断。 |
| 最近确认模式 | 软件最后一次从系统读取或在切换成功后确认的实际模式。 |
| 接管前模式 | 自动化从关闭变为开启时、软件首次写入前读取到的实际模式。 |

系统读取到的实际模式是唯一事实来源；切换请求本身不代表切换成功。

## 3. 自动切换规则

### 3.1 规则优先级

每次评估按以下顺序执行。满足较高优先级后，不再执行后续规则。

1. 自动化关闭、已暂停或因错误停止时，不写入系统电源模式。
2. 发现用户在系统设置或其他软件中修改模式时，记录手动接管；仅当“手动切换后暂停自动化”开启时进入“已暂停”，否则继续按当前实际模式评估。
3. 当前模式为 High Power、但当前环境已不支持 High Power 时，目标模式为 Automatic。
4. 当前模式为 High Power 且“自动退出 High Power”关闭时，保持 High Power。
5. CPU 严格大于 60% 时，目标模式为 High Power（可用时）或 Automatic（不可用时）；无论空闲多久均不得选择 Low Power。
6. CPU 不高于 60% 且已达到空闲时间时，目标模式为 Low Power。
7. CPU 不高于 60% 且尚未达到空闲时间时，目标模式为 Automatic。

用户从 Low Power 恢复操作时，应立即重新评估上述规则：高负载时选择 High Power 或 Automatic；其他情况选择 Automatic。不得继续停留在 Low Power。

### 3.2 切换执行

- 仅当目标模式与系统实际模式不同时才请求切换；同一目标模式不得重复切换。
- 同一时间只允许一次切换。切换完成后必须重新读取并确认系统实际模式。
- High Power 不可用时，不得发送 High Power 切换请求。
- High Power 可用性变化只影响 High Power 的选择，不改变 Low Power 的既有判定条件。
- 仅在“退出低电量模式后恢复亮度”开启时，软件请求进入 Low Power 前读取并暂存当前内建屏幕的用户亮度；只有电源模式回读确认为 Low Power 后，该快照才成为待恢复值。
- 从 Low Power 切换到 Automatic 或 High Power 并回读确认成功后，按用户设定的等待时间恢复进入前的亮度。亮度接口不可用或恢复失败时，不得影响已经确认的电源模式结果，也不得因此进入错误停止或重试电源切换。

## 4. 首版设置

| 设置项 | 默认值 | 说明 |
|---|---:|---|
| 自动化 | 首次启动关闭 | 用户确认授权后才可开启。 |
| 连续无操作时间 | 5 分钟 | 系统未检测到键盘、鼠标或触控板操作达到该时间后，使用空闲时电源模式；用户可修改。 |
| 活跃时检测间隔 | 5 秒 | 用户可修改，单位可切换为毫秒、秒或分钟。 |
| 空闲时检测间隔 | 1 秒 | 用户可修改，单位可切换为毫秒或秒，不提供分钟。 |
| CPU 判断周期 | 最近 15 秒平均值 | 首版固定，不提供高级设置。 |
| 活跃阈值 | 严格大于 60% | 60.0% 不算高于 60%。 |
| 自动退出 High Power | 关闭 | 已处于 High Power 时，不因 CPU 降低或用户空闲而退出；High Power 失去可用性时除外。 |
| 手动切换后暂停自动化 | 关闭 | 检测到用户或其他软件修改电源模式时，是否暂停自动化。关闭时继续按当前实际模式评估，但关闭/退出时仍不覆盖手动选择。 |
| 退出低电量模式后恢复亮度 | 开启 | 关闭时不采集也不写入屏幕亮度。 |
| 亮度恢复等待时间 | 0 毫秒 | 可输入 0–1000 毫秒；如果没有恢复，可以延长等待时间。 |
| 界面语言 | English；系统首选语言为中文时首次默认中文 | 可选择 `English` 或 `中文`。用户手动选择后应持久保存，后续系统语言变化不得覆盖该选择。 |

## 5. 用户手动修改与状态恢复

### 5.1 用户手动修改

当软件没有正在执行的切换、而系统实际模式与最近确认模式不一致时，视为用户在系统设置或其他软件中手动修改。

- 开启“手动切换后暂停自动化”时，立即停止自动写入，状态设为“已暂停”，且不立即改回。
- 进入暂停后，菜单栏显示“已暂停”和“恢复自动”按钮。
- 关闭该设置时，继续读取当前实际模式并按第 3 节评估；无论是否暂停，都记录手动接管，关闭或退出时不得覆盖用户的当前模式。

### 5.2 自动化关闭与退出

- 自动化开启时，在首次写入前保存接管前模式。
- 用户关闭自动化或软件正常退出时，仅当未检测到手动修改且系统当前模式仍等于最近确认模式，才恢复接管前模式。
- 如果上述安全恢复会让系统退出 Low Power，同时亮度恢复仍开启且存在由软件保存的亮度快照，则在电源模式确认成功并经过设定等待时间后恢复该亮度。
- 如果用户期间手动修改过，或无法确认当前模式，不得覆盖系统当前设置。

## 6. 菜单栏界面与错误处理

首版菜单栏只显示：

- 当前电源模式。
- 自动化运行状态。
- 最近一次切换原因。
- 自动化开关。
- 空闲时间设置。
- 活跃时和空闲时检测间隔输入框；活跃时提供毫秒/秒/分钟，空闲时提供毫秒/秒。
- “自动退出 High Power”开关。
- “手动切换后暂停自动化”开关。
- “退出低电量模式后恢复亮度”开关。
- 亮度恢复等待时间输入框。
- 界面语言选择器（`English` / `中文`）；首次默认英文，系统首选语言为中文时首次默认中文。
- “恢复默认设置”按钮；确认后恢复本节所有规则与选项，但不改变当前自动化开关状态。
- “恢复自动”按钮。
- 退出软件。

权限不足、读取失败或切换失败时，停止自动写入并显示原因；不得持续重试或反复请求授权。亮度读取或恢复属于尽力而为，不改变自动化状态。仅记录最近 20 次**自动切换**，每条只包含时间、旧模式、新模式和原因。不得制作复杂的多页设置、图表、详细日志或面向用户的技术信息。

## 7. 实现与验证要求

实现时应把“读取实际状态 → 按第 3 节得出唯一目标模式 → 必要时单次切换 → 回读确认”作为完整的一次决策流程。不得以缓存的目标模式替代系统实际状态。亮度快照只能附着在已确认的 Low Power 周期上，不能代替或改变该事务的系统事实源。

交付前至少验证以下内容：规则边界、High Power 可用性、手动修改后的暂停、失败后的停止写入、接管前模式恢复、并发切换保护，以及语言的系统初始值与用户选择持久化。验证应直接覆盖本节和第 8 节的可观察结果；若实现受 macOS 权限或硬件能力限制，应报告限制与实际检测结果，而非以猜测替代验证。

## 8. 验收标准

| 测试场景 | 预期结果 |
|---|---|
| CPU 60.1%，用户已长时间无操作 | 判定高负载；进入 High Power 或 Automatic，不进入 Low Power。 |
| CPU 60.0%，用户已空闲 5 分钟 | 进入 Low Power。 |
| CPU 40%，用户仍在操作 | Automatic。 |
| 当前 High Power，自动退出关闭，环境仍支持 High Power | 保持 High Power。 |
| 当前 High Power，自动退出开启，CPU 40%，用户仍在操作 | Automatic。 |
| 当前 High Power，自动退出开启，CPU 40%，用户已空闲 5 分钟 | Low Power。 |
| CPU 70%，但当前环境不支持 High Power | Automatic，不发送 High Power 切换请求。 |
| 当前 High Power，当前环境不再支持 High Power | Automatic。 |
| 用户手动修改系统模式，暂停设置关闭 | 继续自动控制，按当前实际模式重新评估；关闭或退出时不覆盖用户选择。 |
| 用户手动修改系统模式，暂停设置开启 | 暂停自动控制，不立即改回；只有点击“恢复自动”后才重新评估。 |
| 权限、读取或切换失败 | 停止自动写入，显示错误，不反复重试。 |
| 目标模式与系统实际模式相同 | 不发送切换请求。 |
| 亮度恢复开启，软件从非 Low Power 自动进入 Low Power | 切换前保存可读取的内建屏幕亮度；模式未确认成功时不把快照视为有效待恢复值。 |
| 亮度恢复关闭，软件进入和退出 Low Power | 不读取、不写入屏幕亮度。 |
| 用户输入亮度恢复等待时间 | 保存值限制为 0–1000 毫秒；首次默认 0 毫秒；如果没有恢复，可延长等待时间。 |
| 首次使用检测间隔设置 | 活跃时显示 5 秒，空闲时显示 1 秒。 |
| 用户切换检测间隔单位 | 活跃时可选毫秒、秒或分钟，空闲时可选毫秒或秒；实际检测时长保持不变。 |
| 首次启动且系统首选语言为中文 | 界面语言默认中文。 |
| 首次启动且系统首选语言不是中文 | 界面语言默认 English。 |
| 用户手动选择界面语言后重新启动 | 保留用户选择，不因系统语言变化而覆盖。 |
| 用户恢复操作，软件确认从 Low Power 切回 Automatic 或 High Power | 经过用户设定的等待时间后，尽力恢复进入 Low Power 前的内建屏幕亮度。 |
| 亮度接口不可用或恢复失败 | 电源模式切换仍按原结果完成，不进入错误停止，不重复切换。 |
| 关闭自动化或正常退出，且未发生手动修改 | 恢复接管前模式。 |
| 关闭自动化或正常退出前发生手动修改 | 不覆盖用户的当前模式。 |
